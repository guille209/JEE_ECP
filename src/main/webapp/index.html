La primera vez que se ejecuta la aplicación, el archivo persistence.xml incluye la generación de tablas automatica mediante el drop-and-create-tables porque sino saltará una excepción al no encontrarse las tablas, una vez ejecutado, la base de datos se completa con la tabla tema y voto.En posteriores ejecuciones dicha propiedad se puede poner a none para que no genere de nuevo las tablas en cada ejecución y borre los datos anteriores.
